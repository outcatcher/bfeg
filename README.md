# bfeg — Big Fucking Excel Generator

Генератор `.xlsx`-файлов произвольного размера со случайными данными.

Полезен для нагрузочного тестирования, бенчмаркинга парсеров Excel и прочих
ситуаций, когда нужен здоровенный файл с правдоподобной таблицей.

## Сборка

```bash
cargo build --release
```

Бинарник: `target/release/bfeg`.

## Использование

```bash
# Файл ~128 МБ (по умолчанию)
bfeg

# Файл на 512 МБ
bfeg --size 512M

# Файл на 2 ГБ с указанием выходного пути
bfeg --size 2G --output huge.xlsx

# Кастомное количество колонок (по умолчанию 100)
bfeg --size 256M --cols 50
```

### Параметры

| Флаг | По умолчанию | Описание |
|---|---|---|
| `--size` | `128M` | Целевой размер файла. Поддерживаются суффиксы `K`, `M`, `G`. |
| `--output` | `random_data.xlsx` | Путь к выходному файлу. |
| `--cols` | `100` | Количество колонок (минимум 4). |

### Типы генерируемых данных

| Тип | Вероятность | Пример |
|---|---|---|
| Целое число | 20% | `-542310` |
| Дробное число | 20% | `3.1415` |
| Строка | 30% | `aBcDeFg123` |
| Дата | 10% | `2025-07-08` |
| Булево | 10% | `TRUE` / `FALSE` |
| Пустая ячейка | 10% | — |

### Как работает

1. **Калибровка** — пишет небольшой сэмпл в `/tmp/bfeg_calib.xlsx` и замеряет
   средний размер в байтах на ячейку.
2. **Расчёт** — по целевому размеру и байтам на ячейку вычисляется нужное
   количество строк. Добавляется 5 % запаса против случайной дисперсии.
3. **Генерация** — пишет `cols × rows` ячеек со случайными данными в один
   потоковый проход.
